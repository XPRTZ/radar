We believe in clean code and <a href="https://docs.microsoft.com/en-us/dotnet/csharp/language-reference/proposals/csharp-10.0/globalusingdirective">global using definitions</a> introduced with C# 10 has given us some more control on cleaning a lot of boilerplate usings that are cluttering up the code files. By defining commonly used usings as global in a single code file, they no longer need to be defined in each single code file individually.
